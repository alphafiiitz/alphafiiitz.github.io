以太坊开发者文档04 智能合约剖析

智能合约剖析

一个智能合约是运行在以太坊上一个地址处的程序。它们由数据和在收到交易时执行的函数组成。以下是构成智能合约的概述。

先决条件

确保您首先阅读了有关智能合约的内容。本文档假设您已经熟悉 JavaScript 或 Python 等编程语言。

数据

任何合约数据都必须分配到一个位置：storage 或 memory。修改智能合约中的存储成本很高，因此您需要考虑数据应该存放在哪里。

Storage

持久性数据称为 storage，由状态变量表示。这些值永久存储在区块链上。你需要声明类型，以便合约可以跟踪它在编译时需要多少区块链上的存储空间。

如果您已经编写过面向对象的语言，那么您可能会熟悉大多数类型。但是，如果您不熟悉以太坊开发，address 对您来说应该是新的。

address 类型可以保存相当于 20 字节或 160 位长的以太坊地址。它以带有前导 0x 的十六进制表示法返回。

其他的类型包括：

  • boolean (布尔)
  • integer (整型)
  • fixed point numbers (定点数)
  • fixed-size byte arrays (固定大小字节数组)
  • dynamically-sized byte arrays (动态大小字节数组)
  • Rational and integer literals (有理字面量和整型字面量)
  • String literals (字符串字面量)
  • Hexadecimal literals (十六进制字面量)
  • Enums (枚举)

有关更多解释，请查看文档：

  • See Vyper types
  • See Solidity types

Memory

仅在合约函数执行的生命周期内存储的值称为内存变量。由于这些不会永久存储在区块链上，因此使用起来要便宜得多。

在 Solidity 文档中了解有关 EVM 如何存储数据（Storage、Memory 和 Stack）的更多信息。

环境变量

除了你在合约中定义的变量外，还有一些特殊的全局变量。它们主要用于提供有关区块链或当前交易的信息。

例如：

Prop State variable Description
block.timestamp uint256 Current block epoch timestamp
msg.sender address Sender of the message (current call)

函数

用最简单的术语来说，函数可以获取信息或设置信息以响应传入的交易。

有两种类型的函数调用：

  • internal - 不创建 EVM 调用

    • internal 函数和状态变量只能在内部访问（即从当前合约或从它派生的合约）

  • external - 创建 EVM 调用

    • external 函数是合约接口的一部分，这意味着它们可以被其他合约和通过交易调用。外部函数 f 不能在内部调用（即 f() 不起作用，但 this.f() 起作用）

它们也可以是 public 或 private

  • public 函数可以在合约内部调用，也可以通过消息在外部调用
  • private 函数仅在定义它们的合约中可见，在派生合约中不可见

函数和状态变量都可以设为公有或私有。

在合约上使用函数更新状态变量：

  • 将 string 类型的参数 value 传入函数：update_name
  • 它被声明为 public，这意味着任何人都可以访问它
  • 它没有被声明为 view，所以它可以修改合约状态

View functions

这些函数承诺不会修改合约数据的状态。常见的例子是“getter”函数——例如，您可以使用它来接收用户的余额。

什么被认为是修改状态：

  1. 写入状态变量
  2. 触发事件
  3. 创建其他合同
  4. 使用 selfdestruct
  5. 通过调用发送以太币
  6. 调用任何未标记为 view 或 pure 的函数
  7. 使用低级调用
  8. 使用包含某些操作码的内联汇编

Constructor functions

构造(constructor)函数仅在首次部署合约时执行一次。与许多基于类的编程语言中的构造函数一样，这些函数通常将状态变量初始化为其指定值。

Built-in functions

除了您在合约中定义的变量和函数外，还有一些特殊的内置函数。最明显的例子是：

  • address.send() - Solidity
  • send(address) - Vyper

这些函数允许合约发送 ETH 给其他账户。

编写函数

你的功能需要：

  • 参数变量和类型（如果它接受参数）
  • internal/external 声明
  • pure/view/payable 声明
  • 返回类型（如果它返回一个值）

事件和日志

事件让您可以从前端或其他订阅应用程序与您的智能合约进行通信。当交易被挖掘(minded)时，智能合约可以发出事件并将日志写入前端可以处理的区块链。
